TANGRAM: Making Unstructured KV Cache Practical for Long-Term Conversation LLM Serving

Hanyang University [cite: 3]

Abstract

As Large Language Models (LLMs) evolve into persistent personal assistants, the ability to maintain Long-Term Conversation (LTC) is critical[cite: 7]. However, preserving this extensive history causes the Key-Value (KV) cache to scale linearly, imposing significant pressure on GPU memory[cite: 8].

While non-uniform KV cache compression preserves accuracy, it disrupts the uniform memory layout relied upon by modern serving systems, leading to severe memory fragmentation[cite: 10, 11].

This paper presents TANGRAM, a novel LLM serving system[cite: 12]. TANGRAM addresses inefficiencies through three core techniques: (1) Profile-Guided Fixed Allocation to eliminate scheduling stalls[cite: 13]; (2) Head-Grouped PagedAttention to resolve memory fragmentation [cite: 14]; and (3) Ahead-of-Time (AOT) Load Balancing to ensure uniform GPU utilization[cite: 15].

Key Contributions

Rethinking KV Stability

We identify that unstructured retention patterns are model-intrinsic and stable[cite: 43]. TANGRAM leverages this to convert dynamic compression into a deterministic contract, eliminating scheduling uncertainty[cite: 44].

Head-Grouped PagedAttention

By decoupling the monolithic page table into localized head groups, we ensure that allocated memory pages align more closely with the practical needs of heterogeneous heads[cite: 45, 46].

AOT Load Balancing

Leveraging static budgets, our offline partitioning algorithm generates a Workload Partition Map[cite: 69]. This eliminates prohibitive runtime planning costs while ensuring perfectly balanced execution across GPU SMs[cite: 70].

Performance Results

TANGRAM improves throughput by up to 2x at the same memory footprint and up to 9x when combined with Unstructured KV Eviction[cite: 16, 783].
