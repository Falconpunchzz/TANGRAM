TANGRAM: Making Unstructured KV Cache Practical for Long-Term Conversation LLM Serving

Hyungmin Kim*, Minsoo Kim*, Jungwook Choi
Hanyang University [cite: 3]
*Equal contribution [cite: 17]
Paper Code

Abstract

As Large Language Models (LLMs) evolve into persistent personal assistants, maintaining Long-Term Conversation (LTC) is critical for personalized user experiences[cite: 7]. However, preserving extensive history causes the Key-Value (KV) cache to grow linearly, creating significant GPU memory and bandwidth pressure[cite: 8].

While non-uniform KV cache compression preserves accuracy by using heterogeneous budgets, it disrupts the uniform memory layout of modern serving systems[cite: 10]. This leads to severe memory fragmentation under paged allocation, preventing theoretical savings from becoming actual memory reductions[cite: 11].

This paper presents TANGRAM, a novel serving system designed to make non-uniform KV caches practical[cite: 12]. TANGRAM addresses inefficiencies through three core techniques: (1) Profile-Guided Fixed Allocation to eliminate scheduling stalls[cite: 13]; (2) Head-Grouped PagedAttention to resolve fragmentation without metadata overhead [cite: 14]; and (3) Ahead-of-Time (AOT) Load Balancing to ensure uniform GPU utilization[cite: 15].

System Features

Eliminating Uncertainty

TANGRAM identifies that head-wise importance patterns are stable and model-intrinsic[cite: 43]. Profile-Guided Fixed Allocation converts dynamic compression into a deterministic contract, allowing the scheduler to plan resources precisely and eliminate prefill stalls[cite: 303, 304].

Resolving Fragmentation

By partitioning heads into distinct groups, Head-Grouped PagedAttention eliminates the "structural dead space" found in coarse-grained page tables[cite: 306]. A Vectorized Block Table ensures this efficiency doesn't create CPU bottlenecks[cite: 47, 48].

AOT Load Balancing

TANGRAM leverages static budget profiles to precompute an optimal Workload Partition Map[cite: 15, 69]. This ensures balanced execution across GPU SMs without the 15-20% latency penalty associated with runtime planning[cite: 70, 298].

Evaluation Results

Our evaluation on multi-turn workloads demonstrates that TANGRAM provides significant gains over baselines like vLLM[cite: 16]:

  • Throughput: Up to 2x improvement at the same memory footprint[cite: 16, 782].
  • Efficiency: Up to 9x improvement when combined with Unstructured KV Eviction[cite: 16, 783].
  • Accuracy: Preserves conversational accuracy on par with dynamic methods[cite: 16, 692].
