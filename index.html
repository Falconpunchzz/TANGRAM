TANGRAM: Making Unstructured KV Cache Practical for Long-Term Conversation LLM Serving

Hyungmin Kim*, Minsoo Kim*, Jungwook Choi
Hanyang University
*Equal contribution [cite: 17]
Paper Code

Abstract

As Large Language Models (LLMs) evolve into persistent personal assistants, the ability to maintain Long-Term Conversation (LTC) has become critical for delivering consistent, personalized user experiences[cite: 7]. However, preserving extensive dialogue history causes the Key-Value (KV) cache to scale linearly, imposing significant pressure on GPU memory[cite: 8].

While non-uniform (unstructured) KV cache compression preserves accuracy by allocating heterogeneous budgets, it leads to severe memory fragmentation under paged allocation, preventing theoretical savings from translating into actual gains[cite: 10, 11].

This paper presents TANGRAM, a novel LLM serving system designed to make non-uniform KV caches practical[cite: 12]. TANGRAM addresses inefficiencies through three core techniques: (1) Profile-Guided Fixed Allocation to eliminate scheduling stalls, (2) Head-Grouped PagedAttention to resolve memory fragmentation, and (3) Ahead-of-Time (AOT) Load Balancing to ensure uniform GPU utilization[cite: 13, 14, 15]. TANGRAM improves throughput by up to 2x at the same memory footprint and up to 9x when combined with unstructured eviction[cite: 16].

Core Methodologies

Profile-Guided Fixed Allocation

We identify that unstructured retention patterns are model-intrinsic and stable regardless of input variations[cite: 43, 348]. By converting dynamic compression into a deterministic "Static Budget Allocation," TANGRAM eliminates scheduling uncertainty and prefill stalls[cite: 38, 44].

Head-Grouped PagedAttention

To resolve fragmentation, we decouple the monolithic page table into independent head groups[cite: 45]. This ensures that allocated memory pages align more closely with the practical needs of heterogeneous heads, reclaiming "structural dead space"[cite: 46, 212].

Ahead-of-Time (AOT) Load Balancing

Leveraging static budgets, we precompute a Workload Partition Map offline[cite: 69]. This eliminates prohibitive runtime planning costs (which can consume 15-20% of iteration time) while ensuring perfectly balanced execution across GPU SMs[cite: 70, 298].

Performance Results

TANGRAM maintains conversational accuracy on par with state-of-the-art dynamic methods while delivering significantly higher system efficiency[cite: 692, 694].

  • Throughput: Up to 9x improvement over vLLM baselines under aggressive compression[cite: 749, 783].
  • Memory: Supports nearly 4x more concurrent requests within the same physical memory footprint[cite: 752].
  • Latency: Zero runtime scheduling overhead for load balancing[cite: 495].
